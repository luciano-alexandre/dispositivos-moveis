# Encontro 14 - Avaliação da Unidade 1

## Finalidade

- Verificar autonomia do estudante na construção de interfaces e fluxos de navegação.
- Medir domínio técnico e organização do código.

## Sugestão de formato

- Parte prática individual.
- Tempo total: 90 minutos.
- Entrega via repositório ou arquivo compactado.

## Competências avaliadas

- Estruturação de projeto.
- Construção de componentes.
- Uso de estado e eventos.
- Implementação de formulários, listas e navegação.

## Exemplo de desafio

Construir um app simples de catálogo acadêmico com:

- tela inicial;
- lista de disciplinas;
- formulário de cadastro;
- navegação entre telas;
- validação mínima.

## Critérios de correção

- Funcionamento.
- Clareza do código.
- Organização dos componentes.
- Consistência visual.

## Pós-avaliação

- Reservar os minutos finais para orientar a autoavaliação.
- Registrar dificuldades recorrentes para retroalimentar a Unidade 2.
