# Encontro 28 - Multiplataforma: React Native e alternativas

## Objetivos

- Comparar abordagens multiplataforma.
- Discutir trade-offs entre produtividade, desempenho e acesso nativo.
- Posicionar React Native no ecossistema atual.

## Conteúdo técnico

Este encontro é excelente para amadurecer visão de engenharia. O estudante deve entender que não existe stack universalmente melhor; existe tecnologia mais adequada ao contexto. Comparar React Native, Flutter, Kotlin Multiplatform e desenvolvimento nativo ajuda a desenvolver critério técnico.

## Quadro comparativo sugerido

| Tecnologia | Linguagem | Reuso | Acesso nativo | Curva |
|---|---|---|---|---|
| React Native | JS/TS | Alto | Bom | Moderada |
| Flutter | Dart | Alto | Bom | Moderada |
| Kotlin Multiplatform | Kotlin | Parcial | Alto | Moderada/Alta |
| Nativo | Kotlin/Swift | Baixo | Máximo | Alta |

## Atividade

- Debate técnico em grupos.
- Cada grupo defende uma stack para um cenário de negócio.
